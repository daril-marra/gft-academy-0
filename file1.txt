
Stati dei files in un progetto git: !conflitto!
1.	MODIFIED: files modificati la cui modifica non è ancora stata “committata” all’interno della repository !conflitto!
2.	STAGED: i files modificati possono essere marcati per essere inseriti all’interno del prossimo snapshot che verrà creato col commit, quando succede i file passano in stato staged !conflitto!
3.	COMMITTED: files salvati all’interno della repository attraverso lo snapshot eseguito dal commit. Questi files sono ora considerati come non modificati (unmodified) in quanto la loro modifica è stata registrata nella cronologia di progetto !conflitto!
4.  UNTRACKED: i files in questo stato non sono ancora aggiunti al progetto !conflitto!
 !conflitto!
A questi primi tre stati corrispondono le tre sezioni di un progetto git: !conflitto!
1.	Working Tree (chiamato anche Working Directory o area di lavoro) ove risiedono i files in stato modified assieme alla versione corrente dei files non modificati !conflitto!
2.	Staging Area ove risiedono i file marcati come staged !conflitto!
3.	Cartella .git (la Repository locale) ove risiedono tutti i files committati in ogni versione !conflitto!
 !conflitto!
